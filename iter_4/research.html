Why the agent is not using the described text data

Investigation target: iter_4/. Generated after inspecting the data registry, ingestion helpers, text feature code, strategy harness, autonomous instructions, tests, cache state, and experiment ledger.

Executive finding

The text data is mostly described as available, not actually made operational for the autonomous loop. The repo contains registry entries, small writer stubs, and one FOMC lexicon feature builder, but the main ingestion command only downloads Yahoo prices; no local text artifacts exist in the observed cache; prepare.py only loads prices; train.py receives only a price DataFrame; and the ledger shows every completed idea is a price-momentum variant. The agent is following the easiest executable path: modify train.py inside a price-only backtest API, run tests, and optimize validation CAGR.

What the repository says should exist

program.md gives the agent a public-dataset policy and says text data should be considered. It lists:

  • fomc_text: FOMC statements and minutes text.
  • beige_book_text: Federal Reserve Beige Book text.
  • sec_edgar: SEC EDGAR filing text and metadata.
  • fred_numeric: FRED numeric macro series.
  • datasets/text_features.py: FOMC lexicon features and publication-lagged alignment.

The same instructions also constrain the autonomous loop to edit only train.py, while prepare.py, datasets/*, and ideas/schema.py are fixed files. This means any text usage must already be consumable from train.py without changing the fixed data layer.

What actually exists

1. Dataset registry: metadata only, with an iter_3 cache-root bug

datasets/registry.py defines DatasetSpec entries for prices, FOMC, Beige Book, SEC EDGAR, and FRED. The registry is useful as a catalog, but it does not load data, validate artifact existence, or connect datasets to the backtester.

datasets/registry.py:
CACHE_ROOT = Path.home() / ".cache" / "macroresearch" / "iter_3"
DATASETS = {
  "prices_yahoo": ...,
  "fomc_text": ...,
  "beige_book_text": ...,
  "sec_edgar": ...,
  "fred_numeric": ...,
}

Specificity: in iter_4, the registry still points at ~/.cache/macroresearch/iter_3. The test suite explicitly asserts that stale path via test_iter3_cache_root_is_isolated, so the bug is protected by a passing test. Meanwhile, prepare.py uses ~/.cache/macroresearch/iter_4 for prices. This split makes the registry less trustworthy as the operational source of truth.

2. The public ingestion command ingests only prices

scripts/ingest_public_macro.py imports only datasets.ingest_prices.ingest_prices and calls it for DEFAULT_UNIVERSE + DBMF. It does not call FOMC, Beige Book, SEC, or FRED ingestion.

from datasets.ingest_prices import ingest_prices
from prepare import BENCHMARK_SYMBOL, DEFAULT_UNIVERSE

symbols = list(dict.fromkeys([*DEFAULT_UNIVERSE, BENCHMARK_SYMBOL]))
paths = ingest_prices(symbols)

So the setup command documented in README.md:

uv run scripts/ingest_public_macro.py

does not ingest public macro text. Despite the script name, it is a Yahoo-price ingestion script.

3. Text ingestion modules are mostly artifact writers, not live ingestors

These modules are tested with toy fixtures, which proves deterministic formatting and schema shape, not end-to-end availability for the agent loop.

4. The only implemented text feature builder is FOMC lexicon scoring

datasets/text_features.py implements:

  • tokenize(text): lowercase regex tokenization using [a-z]+(?:-[a-z]+)?. Hyphenated terms stay as one token, so inflation-risk is not counted as inflation or risk.
  • lexicon_score(text, positive_terms, negative_terms): (positive_count - negative_count) / token_count; empty text returns 0.0.
  • build_fomc_features(events): requires published_at and text, sorts events by publication time, and emits token_count, hawkish_minus_dovish, uncertainty_score, source_dataset, feature_version, and lexicon_version.
  • align_event_features_to_prices(prices, event_features): converts published_at to UTC, drops timezone, normalizes to the calendar day, adds one calendar day, forward-fills event values onto the price index, and reindexes to trading days.

The policy-sentiment lexicon is deliberately small:

FeatureTerms / calculation
hawkish_minus_dovishHawkish terms: inflation, restrictive, tightening, elevated, overheating, hike, hikes, firming. Dovish terms: cut, cuts, weakness, unemployment, accommodative, easing, slack, slowdown. Score is net count divided by token count.
uncertainty_scoreUncertainty terms: uncertain, uncertainty, risk, risks, volatile. Score is count divided by token count because the negative set is empty.

Specificity: align_event_features_to_prices includes metadata columns unless the caller removes them. Because source_dataset, feature_version, and lexicon_version are strings, the aligned DataFrame may contain object columns. A strategy author must select numeric columns before using them in rules.

5. The backtest harness is price-only

prepare.py is the fixed data/evaluation harness. Its core loader is load_prices(symbols), which returns a DataFrame of Yahoo adjusted-close prices plus DBMF. The WeightSelector protocol is:

class WeightSelector(Protocol):
    def __call__(self, day: pd.Timestamp, prices: pd.DataFrame) -> Weights: ...

During monthly rebalancing, the harness calls:

history = prices.loc[:previous_day, tradable]
current_weights = validate_weights(select_weights(previous_day, history), tradable_set)

So select_weights receives only historical prices for tradable symbols. There is no load_public_dataset, no load_fomc_events, no event-feature join, and no extra feature store passed into the strategy. The model interface strongly nudges agents toward price-only ideas because price history is the only fully operational input in the fixed harness.

6. train.py uses only price momentum

The current editable strategy imports only pandas, ideas.schema, and prepare. It does not import datasets.text_features, datasets.registry, or any text ingestion module. The active IDEA.feature_inputs are:

('price_momentum_21d', 'score_weighted_top2', 'equity_sector_universe')

The weight function ranks tradable symbols by trailing price return and optionally by trend, volatility, score weighting, SPY guard, and cash fallback. It has no reference to FOMC, Beige Book, EDGAR, FRED, or any aligned event feature.

Cache state

The observed cache has no ~/.cache/macroresearch/iter_4 directory. ~/.cache/macroresearch/iter_3/market exists and contains price CSVs, including DBMF and multiple ETFs. No cached FOMC, Beige Book, SEC, or FRED artifacts were observed in the cache listing. This matches the code: the public ingestion script writes prices only, and the registry points at iter_3.

Root causes

  1. The “available datasets” are not operational datasets. They are registry metadata and writer helpers. The loop has no command that materializes text artifacts from public sources.
  2. The fixed evaluation harness exposes only prices. The autonomous agent is told to edit only train.py, but prepare.py owns data loading and only loads Yahoo prices. This makes text usage optional, manual, and brittle from inside train.py.
  3. The documented setup command is misleading. scripts/ingest_public_macro.py sounds macro-wide but only calls price ingestion.
  4. The registry cache root is stale. datasets.registry.CACHE_ROOT points to iter_3 and a test asserts that behavior. This conflicts with prepare.py, which points to iter_4.
  5. Text features are narrow and only partially implemented. FOMC has simple lexicon features; Beige Book and EDGAR have no feature builders; SEC has metadata only, not filing text content; FRED has URL/writer helpers only.
  6. The agent’s optimization pressure rewards fast price-only iterations. Price momentum variants are trivial to express in train.py, pass tests, and can be evaluated quickly. Text variants require discovering absent artifacts, writing ad hoc loaders, handling object columns, handling missing events, and preserving point-in-time alignment.
  7. No enforcement exists in the rationale gate. IdeaRationale.feature_inputs only requires non-empty strings; it does not require at least one public text dataset trial or validate that named feature inputs were actually computed.

How text data could be used today, without editing fixed files

It is technically possible but awkward. A train.py-only strategy would need to:

  1. Locate a text artifact path itself, because no loader exists. For example, guess ~/.cache/macroresearch/iter_4/fomc/fomc_events.tsv or use datasets.registry.CACHE_ROOT, which currently points at iter_3.
  2. Read events with pd.read_csv(..., sep="\t").
  3. Call build_fomc_features(events).
  4. Inside select_weights, call align_event_features_to_prices(prices, features) on the historical price DataFrame, then inspect the latest aligned numeric row.
  5. Use those latest features as a regime gate or asset-allocation tilt, while still returning only tradable-symbol weights.
  6. Handle missing text artifacts gracefully, otherwise the run crashes and is rejected.

This path is more complex than price-only momentum and is not scaffolded by the repo. Also, because no FOMC event TSV exists in the observed iter_4 cache, step 1 fails unless someone manually creates the artifact outside the fixed autonomous loop.

Important implementation specificities and pitfalls

  • Publication lag: align_event_features_to_prices makes features effective on published_at normalized to date plus one calendar day. A Friday publication becomes Saturday and then forward-fills to the next trading day on reindexing.
  • FOMC minutes date: The instruction says minutes must align to publication date, not meeting date. The current publication_timestamp_from_url just extracts the first YYYYMMDD from the URL. This is safe only if the URL date really is publication date.
  • Time of day: FOMC ingestion writes T00:00:00Z. The aligner then adds one day, preventing same-day lookahead but also ignoring exact release times.
  • String metadata in aligned features: Text feature metadata columns are not automatically stripped before alignment.
  • Hyphenated token behavior: The tokenizer treats inflation-risk as one token, so simple lexicon words do not match inside hyphenated compounds.
  • SEC data is metadata only: The code can write cik, accession_number, form_type, accepted_at, and primary_document_url, but not downloaded filing text or derived sector-level uncertainty features.
  • Beige Book has no features: It can normalize and write documents, but no growth/inflation/labor lexicon or district aggregation exists.
  • FRED is marked not point-in-time safe: The registry sets fred_numeric.point_in_time_safe=False and notes ALFRED vintages should be preferred. The loop does not provide vintage-safe FRED features.

Why the agent behaved this way

The agent is not ignoring a ready-to-use text feed; it is encountering an instruction/code mismatch. The instruction layer says “consider text data,” but the executable layer says “here is a price-only strategy function; edit only this file; run a price-only evaluator; use the ledger row produced by train.py.” Given that environment, the agent’s local optimum is to keep generating price-momentum variants. The text route requires extra discovery, non-obvious cache paths, absent data artifacts, and ad hoc integration code in the strategy file. Nothing in tests, schema, or the loop forces a text trial, and nothing in prepare.py makes text first-class.

Recommended fixes if the goal is actual text-data experimentation

  1. Make text artifacts real. Extend scripts/ingest_public_macro.py or add dedicated scripts that actually fetch and cache FOMC events, Beige Book releases, selected SEC filing text, and vintage-safe macro series. Write manifests with source URLs, hashes, row counts, and point-in-time flags.
  2. Fix cache-root consistency. Change datasets.registry.CACHE_ROOT and its test from iter_3 to iter_4, or derive the iteration root from one shared constant.
  3. Add fixed loaders. Provide functions such as load_fomc_events(), load_fomc_features(prices), and load_public_feature_frame(prices) in fixed data code. Agents should not have to guess artifact paths.
  4. Expose features cleanly to train.py. Either pass an aligned feature frame into select_weights, or provide a documented train-safe helper that returns point-in-time aligned numeric features.
  5. Broaden feature builders. Add Beige Book growth/inflation/labor pressure features, SEC sector uncertainty/risk features, and FRED/ALFRED numeric surprise/regime features if those datasets are advertised.
  6. Make agent instructions mandatory rather than aspirational. Add a loop requirement like “at least one in every N formal trials must use a non-price public dataset unless no artifact exists; if no artifact exists, append a needs_data diagnostic row or report the missing artifact.”
  7. Validate actual feature use. Extend the rationale or run output to record computed feature columns and source artifact paths. This prevents a row from claiming fomc_text in feature_inputs without actually using it.

Bottom line

The repo currently has a text-data promise but not a text-data workflow. The public text datasets are listed and partially scaffolded, but only price data is operationally loaded and optimized by the autonomous loop. Until text ingestion, cache paths, loaders, aligned feature exposure, and loop requirements are made first-class, the agent will continue to discover that price-only momentum is the shortest path to valid, high-validation-CAGR experiments.

Post-implementation status

The recommended fixes have now been implemented in the working tree. The repo has a shared datasets/cache.py iter_4 cache root, real public-data ingestion paths with manifest output and explicit needs_data messages, validated fixed loaders, publication-lagged public feature frames, broadened FOMC/Beige/SEC/FRED feature builders, automatic public feature joining in prepare.py, mandatory feature-usage output in program.md, and a first text-enabled train.py candidate.
